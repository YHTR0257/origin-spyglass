# /explore — コード探索スキル

コード修正の前に、関連ファイルと影響範囲を調査して報告する。
修正は一切行わない。読み取り専用の探索のみ。

## 実行手順

### Step 1: 対象ファイルの特定

`/translate` の出力（論点ごとの対象モジュール）をもとに、関連するファイルを開いて読む。

- 対象として挙げられたファイルをすべて開く
- 各ファイルについて、関連する関数・クラス・行を特定する
- import 先・呼び出し元を1階層たどり、直接の依存関係を把握する

### Step 2: 影響範囲の報告

以下のフォーマットで報告する:

```
## 探索結果

### 論点1: [/translate の論点タイトル]

**直接の対象**:
- `ファイルパス` — 関数名/クラス名（行番号）: 現在の挙動の要約

**影響を受ける可能性のあるファイル**:
- `ファイルパス` — 理由（import している / 呼び出している / 同じスキーマを使っている 等）

**確認事項**:
- [この変更で考慮すべき点、既存テストの有無、未実装部分との関係 等]
```

### Step 3: 修正方針の提案

探索結果をもとに、修正方針を簡潔に提案する。
ユーザーが「それで進めて」と言うまで、コードの修正には着手しない。

## 禁止事項

- ファイルの作成・編集・削除を行わない
- git 操作を行わない
- 推測で影響範囲を語らない。必ずファイルを開いて確認する
- 影響範囲が広い場合は、ユーザーに確認してから次のステップに進む
